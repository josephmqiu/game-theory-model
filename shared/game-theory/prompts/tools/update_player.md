Modify an existing player's attributes by ID; only the fields you provide are changed, all others remain as-is.
Inputs: `id` (required — the player ID to update), plus any fields to change such as name, type, role, objectives, or constraints.
Revising players mid-analysis is expected — objective functions often sharpen when historical research in Phase 4 surfaces new constraints.
If the player was previously reviewed or accepted by the user, use propose_revision instead of this tool so the change goes through the review workflow.

# Session 6.3 — Phase 7: Assumption Extraction and Sensitivity

## Goal
Make every assumption explicit, rate sensitivity, identify correlated clusters, classify game-theoretic vs empirical.

## Depends on
- 6.2 (Phase 6 formalization complete — assumptions extracted from formal models)
- 1.1 (Assumption type with new fields: game_theoretic_vs_empirical, correlated_cluster_id)
- 4.3 (sensitivity analysis from solvers)

## Contract

### Phase 7 Output

```typescript
interface AssumptionExtractionResult {
  phase: 7
  status: PhaseResult
  assumptions: ProposedAssumptionFull[]
  correlated_clusters: CorrelatedCluster[]
  sensitivity_summary: SensitivitySummary
  proposals: ModelProposal[]
}

interface ProposedAssumptionFull {
  temp_id: string
  statement: string
  type: 'behavioral' | 'capability' | 'structural' | 'institutional' | 'rationality' | 'information'
  sensitivity: 'critical' | 'high' | 'medium' | 'low'
  what_if_wrong: string // what happens to prediction if this is wrong
  game_theoretic_vs_empirical: 'game_theoretic' | 'empirical'
  correlated_cluster_id: string | null
  evidence_quality: 'direct_evidence' | 'inference' | 'assumption_only'
  evidence_refs: EntityRef[]
  affected_conclusions: EntityRef[] // scenarios, equilibria that depend on this
  confidence: EstimateValue
}

interface CorrelatedCluster {
  id: string
  label: string // e.g., "Iran regime fragility"
  description: string // why these assumptions move together
  latent_factor: string
  assumption_ids: string[] // temp_ids
  affected_domains: string[] // what changes if the whole cluster shifts
}

interface SensitivitySummary {
  critical_count: number
  high_count: number
  medium_count: number
  low_count: number
  inference_only_critical: number // critical assumptions with only inference support (flag!)
  largest_cluster_size: number
}
```

### Phase 7 Behavior
1. AI uses 'assumption_analysis' role
2. Lists EVERY assumption from Phases 1-6 (playbook 7a) — 6 categories
3. Rates sensitivity: "What happens to prediction if this is wrong?" (playbook 7b)
4. Critical + high assumptions need strongest evidence — flag if inference-only
5. Identifies correlated clusters driven by same latent factor (playbook 7c)
6. Classifies game-theoretic vs empirical (playbook 7d) — this matters for diagnosing prediction failures
7. Uses M4 sensitivity analysis (from 4.3) to inform sensitivity ratings where solver results exist
8. Produces add_assumption command proposals via diff review

### Revalidation Triggers
Phase 7 can trigger:
- `critical_empirical_assumption_invalidated` -> loops back to Phase 7, then re-runs Phase 8 + 9
- Note: Only empirical assumptions trigger revalidation. Game-theoretic assumption failures require a different response (reframing the model structure, not re-running phases).

### Conversation Integration
- "Extracting assumptions... Found 23 assumptions across 6 categories."
- "4 critical assumptions identified. 2 have only inference support — flagging for review."
- "3 correlated clusters found. Largest: 'Iran regime fragility' affects 5 assumptions."
- Inline assumption cards with sensitivity badges

## Behavior rules
- Every assumption must have a "what if wrong" statement
- Critical assumptions without direct evidence support get warning flag in conversation
- Correlated clusters matter for scenarios — flagged for Phase 9 awareness
- Game-theoretic vs empirical classification is metadata, not judgment of quality

## Implementation Deviations

1. **SensitivitySummary renamed to Phase7SensitivitySummary**: Avoids name collision
   with potential M4 solver sensitivity types. Fields are identical to spec.

2. **Owner attachment system**: Implementation adds `OwnerAttachment` types
   (game, formalization, game_node, game_edge, normal_form_estimate, etc.) not in
   the spec. Rationale: assumptions need fine-grained ownership tracking to specific
   payoff cells, nodes, and edges — not just game-level references.

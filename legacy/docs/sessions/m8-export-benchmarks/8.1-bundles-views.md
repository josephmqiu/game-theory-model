## Behavior

### Bundle Export

- Rule: A `.gtbundle` is a zip file containing: the analysis `.gta.json` file, source snapshots, AI run logs, extraction outputs, prompt versions used, optional play sessions, and a `manifest.json` with checksums.
- Rule: Every file in the bundle has a checksum in the manifest. Checksums are computed at export time.
- Rule: Source snapshots are immutable once captured. They are included as-is.
- Rule: Play sessions are optional in the bundle. The export options control what is included.
- Rule: Events from the audit trail are included in the bundle for replay capability.
- Rule: Bundle export includes the SQLite audit trail events as JSON files. On import, these events are loaded into the workspace SQLite database, enabling Diff view and replay. The BundleImportResult should expose the restored event count.

### Bundle Import

- Rule: Import reads the zip, verifies all checksums against the manifest, and reports any failures.
- Rule: If checksums pass, the analysis file is loaded into a new workspace.
- Rule: If checksums fail, the import warns but does not block -- the user decides whether to proceed.
- Rule: Import never overwrites an existing workspace. It creates a new one.
- Example: Export a bundle from workspace A. Import in a fresh workspace B. The model in B matches A exactly. Checksum verification passes.

- Unit test: `importBundle` reads a valid bundle and returns the analysis with `checksum_verification.all_valid === true`.
- Unit test: `importBundle` with a corrupted file reports the checksum failure in the result.
- Unit test: `verifyBundleChecksums` detects tampered files.
- Integration test: Export a bundle from a workspace with play sessions and AI runs. Import in a fresh workspace. The model matches exactly.

## Guidance

- Use Tauri's filesystem APIs for zip creation and extraction. The bundle format should be a standard zip -- no custom binary format.
- Checksums should use SHA-256. Compute them over raw file bytes, not parsed content.
- Do not build the benchmark harness here. That is session 8.2.
- Bundle export/import should be tested with fixture files that include all optional sections (play sessions, AI runs, source snapshots).

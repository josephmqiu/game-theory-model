# Session 7.3 — Play-Out Engine

## Goal
Headless play-out engine for turn-by-turn interactive simulation. This is a FEATURE (not a pipeline phase) -- users explore after analysis is complete.

## Depends on
- 4.1-4.3 (solvers for AI decision-making)
- 5.1 (MCP server — play tools delegate to play engine for AI-controlled players)
- 7.1 (composition engine contracts for cross-game effects during play)

## Preserves from old 7.3
This session preserves ALL contracts from the old `m7-crossgame-playout/7.3-play-engine.md`:
- PlaySession, PlayBranch, PlayTurn, PlayActor types
- SessionStateCheckpoint, BranchComparison types
- AITurnPhase, AITurnProgress types
- PlayEngine class with all methods
- Session lifecycle, branching, forking, quick rewind
- Shock injection mechanism
- Session rebase algorithm
- promote_play_result command

## Changes from old spec
1. Play-out is positioned as a feature accessed from the Play-out screen (sidebar nav), not a pipeline phase
2. Play-out integrates with the 10-phase analysis results: formalizations from Phase 6, scenarios from Phase 9
3. Users can start play-out from any scenario: "Play out Scenario 2: Regional Escalation"
4. AI turn decision-making informed by Phase 6 equilibrium analysis and Phase 7 assumption sensitivities

## Additional contract

```typescript
interface PlayFromScenarioInput {
  scenario_ref: EntityRef
  starting_conditions: string // narrative of starting point
  player_assignments: { player_id: string; policy: 'human' | 'ai' | 'equilibrium' | 'heuristic' }[]

```

## Behavior rules
- Same as old spec: sessions are L3 derived, never in .gta.json
- promote_play_result goes through command spine
- Session rebase on canonical changes
- Play results can feed back into analysis via conversation: "Play-out revealed X -- should we update scenarios?"

# Session 7.4 — Play-Out View

## Goal
React UI wrapping headless play engine. AI turn progress streaming, shock UX, cascade inspection, branch comparison.

## Depends on
- 7.3 (play engine)
- 3.1 (sidebar navigation -- Play-out is a top-level nav item)
- 3.7 (conversation integration -- play results can feed back to Overview)

## Preserves from old 7.4
This session preserves ALL contracts from the old `m7-crossgame-playout/7.4-play-view.md`:
- React hooks: usePlaySession, useAITurnProgress, useBranchComparison, useTurnHistory
- CascadeInspection, ShockProposal, BranchComparisonViewModel types
- AI turn progress streaming
- Shock injection UX
- Cascade inspector
- Branch comparison view
- Mixed human + AI play
- View coordination via bus

## Changes from old spec
1. Play-out screen has inspector panel (320px right side) like all other screens
2. "Start from Scenario" button links to Phase 9 scenarios
3. Play results can be sent back to conversation: "This play-out suggests the escalation scenario is more likely than modeled"
4. Phase progress bar visible at bottom (same as Overview) for context

## Behavior rules
- Same as old spec for play mechanics
- Inspector shows entity detail when clicking players, games, turns in play view
- Conversation integration is optional -- user can play without sending results back

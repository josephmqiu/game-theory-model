# Session 7.2 — Phase 10: Meta-Check and Final Test

## Goal
AI runs the playbook's 10 meta-check questions and 6 final test questions. Includes adversarial challenge (moved from old Phase 7).

## Depends on
- 7.1 (Phases 8-9 complete -- scenarios, thesis, eliminations)
- 6.1 (revalidation mechanism -- meta-check can trigger final revalidation)

## Contract

### Phase 10 Output

```typescript
interface MetaCheckResult {
  phase: 10
  status: PhaseResult
  meta_check_answers: MetaCheckAnswer[]
  final_test_answers: FinalTestAnswer[]
  adversarial_result: AdversarialResult
  revisions_triggered: RevalidationTrigger[]
  analysis_complete: boolean // true if meta-check passes without triggering revisions
}

interface MetaCheckAnswer {
  question_number: number // 1-10
  question: string
  answer: string
  concern_level: 'none' | 'minor' | 'significant' | 'critical'
  revision_triggered: RevalidationTrigger | null
  evidence_refs: EntityRef[]
}

interface FinalTestAnswer {
  question_number: number // 1-6
  question: string
  answer: string
  completeness: 'fully_answered' | 'partially_answered' | 'cannot_answer'
  gap_description: string | null // if partially/cannot, what's missing
}

// Storage: L3 (derived). Stored in SQLite alongside phase execution logs.
// NOT stored in .gta.json — these are AI-generated diagnostic outputs, not canonical model data.
// The 6 Final Test answers are displayed in the Scenarios screen but read from SQLite, not L1.
```

### 10 Meta-Check Questions (from playbook)

1. Which player have I spent the least time analyzing?
2. Which game am I most confident about? (Overconfidence = under-examination)
3. What's the strongest counterargument to my central thesis? (Steel-man the opposition)
4. What information would most change my predictions?
5. Which claim is most dependent on discretionary judgment rather than formal structure?
6. Have I treated a public statement as more informative than it deserves?
7. Have I treated a stated red line as lexicographic without evidence?
8. Did I add complexity because the event is complex, or because I was reluctant to simplify?
9. Could a smaller model explain 80% of the behavior just as well?
10. Which adjacent analytical tool did I use, and did I label it correctly?

### 6 Final Test Questions (from playbook)

1. What is the smallest game that captures the core strategic tension?
2. What did the repeated interaction history do to today's incentives and beliefs?
3. Which strategies are actually feasible once institutional, domestic, and escalation constraints are added?
4. What equilibrium or near-equilibrium behavior follows from that structure?
5. Which parts of the final forecast come from the model, and which come from analyst judgment?
6. What specific evidence would change the central prediction?

### Adversarial Challenge

```typescript
interface AdversarialResult {
  challenges: AdversarialChallenge[]
  overall_assessment: 'robust' | 'defensible' | 'vulnerable' | 'flawed'
}

interface AdversarialChallenge {
  id: string
  category: ChallengeCategory
  target: EntityRef | null
  severity: 'critical' | 'high' | 'medium' | 'low'
  challenge: string
  evidence_against: EntityRef[]
  suggested_revision: string
  affected_conclusions: EntityRef[]
  response_status: 'unaddressed' | 'accepted' | 'rejected' | 'deferred'
  analyst_response: string | null
}

type ChallengeCategory =
  | 'framing_choice'
  | 'omitted_actor'
  | 'omitted_strategy'
  | 'omitted_latent_driver'
  | 'overconfident_payoff'
  | 'naive_independence'
  | 'unjustified_equilibrium'
  | 'linear_scenario'
  | 'missing_cross_game_effect'
  | 'evidence_quality'
  | 'structural_bias'
```

### Phase 10 Behavior
1. AI uses 'meta_check' role (preferably different model from earlier phases for fresh perspective)
2. Answers all 10 meta-check questions with concern level
3. Answers all 6 final test questions with completeness assessment
4. Runs adversarial challenge: structured critiques targeting specific entities
5. If meta-check reveals significant concerns -> trigger revalidation (back to earliest affected phase)
6. If all questions answered satisfactorily -> analysis marked complete
7. Final test questions displayed prominently in Scenarios screen

- User can respond to challenges: "I disagree with challenge #2 -- here's why..."

## Behavior rules
- Meta-check must be honest -- AI should find real concerns, not rubber-stamp
- Adversarial challenge uses different model if available (cross-model adversarial is more effective)
- Challenges stored in L3 (derived), not canonical -- user responds and accepts/rejects
- Analysis is NOT complete until meta-check passes without triggering revisions
- `analysis_complete` MUST be `false` if any `FinalTestAnswer.completeness` is `'cannot_answer'`. The playbook says "If the methodology cannot answer those six questions, it is not yet finished."
- `analysis_complete` SHOULD be `false` if any is `'partially_answered'`, but this can be overridden by user (who may accept partial answers for simpler analyses).
- Final test answers persist and display in Scenarios screen
